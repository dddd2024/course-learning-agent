本轮按“对用户好用”而不是“功能能跑”的标准，依据《大型程序设计实践 2026》课程要求，对 test 账户已有数据完成桌面端与 390px 手机端的端到端审计，并实施了跨导航、课程、资料、学习、问答、计划、待办、测验、知识图谱和个人设置的大范围体验改造。

## 主要改进

1. 全局导航改为分组信息架构，补齐手机抽屉、遮罩、焦点管理、跳到主内容、面包屑和页面标题。
2. 仪表盘允许主动切换课程；部分接口失败时保留其他模块，避免整页一起空白。
3. 课程卡片、资料、待办和测验在手机端改成可读卡片或自适应布局，不再把桌面表格硬塞进窄屏。
4. 学习页增加正文/图片移动端切换、完整文本块翻页，并把自动知识点生成改成用户明确触发。
5. 问答页增加移动端会话抽屉和可恢复会话；“重新生成”改为诚实的“再次提问”，避免暗示覆盖原回答。
6. 计划增加列表与详情持久化，课程入口会保留课程上下文；多课程计划使用清晰的优先级输入。
7. 待办补齐分页、暂停/继续、完成后撤销，并在状态变更时保持时间字段一致。
8. 测验区分草稿与已提交状态，已提交记录可进入逐题复盘；错误知识点计数按单次测验去重。
9. 知识图谱修复异步详情串台和关系修改后的陈旧状态，并提供屏幕阅读器可访问的文本摘要。
10. 个人中心只有在连接测试成功后才能启用配置；注册确认密码仅在注册模式出现。
11. 资料检索移除容易被误解为答案可信度的百分比“相关度”。

## 剩余优化项

- P2 性能：主入口包约 1.11 MB，知识图谱包约 663 KB；应继续拆分 Element Plus/ECharts 等重依赖并按路由延迟加载。
- P2 移动端：日志和 Agent 运行属于开发诊断界面，当前依赖局部横向滚动；后续可改为摘要卡片 + 抽屉详情。
- P2 无障碍：本轮验证了语义标签、焦点入口和基本键盘结构，但没有完成真实屏幕阅读器、200% 缩放和系统高对比模式认证。
- P2 Agent 质量：本轮为避免产生额外模型费用或污染 test 账户历史，没有发送新的付费模型提问；验证基于既有成功运行、历史消息、引用数据和接口契约，不等同于对回答正确率的盲测。
